项目概览

Long-Term Person Re-Identification 的难点在于真实世界不会保持同一套衣服、姿态和活动。传统 ReID 常假设短时间、衣服不变；cloth-changing ReID 又常局限在 walking / standing。长期 ReID 更接近现实：同一个人可能换衣服、换活动、换视角，外观相似的人也可能造成混淆。

ICCV 2023 论文提出 3DInvarReID，用 joint two-layer implicit model 分离 identity、pose、clothed shape 和 texture，同时学习 naked body shape feature。我在 2023 暑期 Paid RA 中负责 long-term person ReID 数据集设计与整理，组织 20,000+ 跨服装/跨活动人体图像，为 3D shape representation 和论文实验提供数据基础。

1. 问题： 衣服不是身份

许多 ReID 模型会无意中把衣服纹理当成身份捷径。短期监控场景里这个捷径有时有效，但长期场景下同一个人会换衣服，不同人也可能穿相似服装。只靠 appearance feature 会导致模型在真实时间跨度里失效。

3DInvarReID 的思路是把身份线索转向更稳定的人体形状表示，同时显式建模 clothed shape 和 texture。这个方向要求数据本身覆盖足够多的活动和服装变化，否则评测仍会被简单外观模式支配。

2. My Role: 数据边界和实验口径

我的工作不是主导模型方法，而是支持数据基础：设计和整理 long-term ReID 数据，组织 20,000+ 跨服装/跨活动人体图像。这里的关键不是简单收集更多图片，而是控制哪些变化属于身份无关因素、哪些场景会让模型过拟合、哪些图像能构成有效 query / gallery。

这段经历训练了我对 eval 的敏感度。后来做 patent search 和 agent eval 时，我同样关注 answer-blind、heldout、failure mode 和 rubric，因为没有清晰数据边界，任何“准确率提升”都可能只是口径变化。

3. Paper Context

论文构建了 CCDA dataset，并在 Celeb-reID、PRCC、LTCC、CCVID、Market-1501、MSMT17 和 HBW reconstruction 等数据上评估。CCDA 包含 100 subjects、1,555 images，并区分 normal / challenging query。

论文报告中，3DInvarReID 在 CCDA 上达到 mAP 21.7、Rank1 11.1、Rank5 30.5，高于 ReIDCaps 和 CAL；ablation 显示 full model Rank1 36.3，w/o 3D 为 28.4，w/o pre-training 为 29.3，w/o 3D clothing 为 32.1。HBW reconstruction CD-L2 0.610，优于 LVD 和 SHAPY。

4. Connection to Later Work

这不是 agent 项目，但它解释了我为什么在 Lawless 里强调 eval loop。无论是 ReID 还是 legal AI，系统都可能学会错误捷径：ReID 可能学衣服，patent search 可能学品牌关键词，TRO assistant 可能学固定模板。真正的工程工作是设计数据和评测，让系统必须面对目标任务本身。

和普通 ReID 数据工作的区别

普通 ReID 很容易在短时间、同服装、同场景上取得好指标，但长期身份识别真正困难的是衣服、姿态、活动和视角变化。这个项目的区别在于围绕 long-term identity problem 组织数据，避免模型只学衣服纹理或背景捷径。

我的贡献边界是数据集设计与整理，而不是夸大成方法主导。它的价值在于支持 3D shape representation 这类更稳定身份线索的评估，也训练了我后续做 agent eval 时对 dataset split、failure mode 和 shortcut 的敏感度。

这里的 20,000+ 指 RA 阶段参与整理/筛选的跨服装、跨活动候选图像基础；论文公开的 CCDA 版本保留 100 subjects / 1,555 images。这个边界要写清楚，否则读者会把 raw curation scale 和 published dataset size 混在一起。

5. 技术补充： 数据集为什么是方法的一部分

在 Long-Term ReID 里，数据集不是附属材料，而是定义任务本身。若 query/gallery 只覆盖同一服装，模型学衣服也能拿高分；若活动太单一，模型可能学姿态或场景；若同一人样本太少，3D shape representation 很难被稳定评估。

因此整理 20,000+ 跨服装/跨活动人体图像时，重点是让变化因素足够丰富，同时保持身份标注和实验口径可用。哪些图像属于同一身份、哪些变化应该被视为身份无关、哪些条件构成 challenging query，这些决定了模型是否真的面对 long-term identity problem。

这类数据工作看起来不如模型名字显眼，但它常常决定论文实验是否有意义。对我来说，它也是后来设计 agent eval 的基础训练：先定义任务，再定义数据，再看系统是否真的解决了这个任务。

6. 如何理解结果

3DInvarReID 的结果不能只读成一个 Rank1 数字。它的关键是把 clothing/pose/texture 等非身份因素从 identity shape feature 中分离，让长期匹配不再过度依赖外观。ablation 里去掉 3D 或 pre-training 后性能下降，说明 3D 表示和训练流程都在贡献。

这个项目也提醒我：模型要泛化，必须避免容易但错误的 shortcut。Lawless patent search 里的 brand-only shortcut、TRO 里的模板化回复，本质上是同一类问题。系统要被设计到不能靠捷径拿分。

7. 数据补充： 从图像收集到任务定义

20,000+ 图像的整理工作可以进一步拆成几类判断：同一身份是否跨越足够多服装和活动；图像质量是否足够支持 3D shape 或人体特征分析；query/gallery 的设置是否会让模型靠背景或衣服取巧；不同数据来源之间是否存在标注和分布差异。

这些问题在论文里通常被压缩成 dataset construction，但对实际研究很关键。数据集如果没有覆盖目标变化，模型再复杂也只是在错误任务上优化。长期 ReID 的价值恰好在于迫使模型面对“身份不等于衣服”这个事实。

8. 补充说明： 从 ReID 到 agent eval 的迁移

ReID 数据集工作给我的一个长期影响是：评测口径本身就是产品的一部分。一个系统在错误数据上拿高分，可能比低分更危险，因为它会让团队以为问题已经解决。长期 ReID 里，如果衣服、背景或姿态泄漏身份，模型就会学到错误捷径；法律 agent 里，如果 benchmark 只检查是否写了流畅报告，agent 就会学会写得像专家但不一定召回关键来源。

因此我在 Lawless 里很强调 answer-blind runner、heldout cases、grader rubric 和 failure taxonomy。它们和视觉研究里的 dataset split、ablation、metric design 是同一类工程判断。区别只是对象从图像模型变成了 agent skill。

这段经历的贡献边界很清楚：我不是 3DInvarReID 的一作或方法主导，而是参与数据基础。这个边界让项目表达更可信，也更接近真实研究分工。

9. 补充说明： 研究训练如何迁移到产品

ReID 展示了研究训练和产品训练之间的连续性。Lawless 强调 evidence、eval 和 failure mode；这些不是凭空出现的产品词，而是来自更早的视觉研究经验。做数据集时要问“模型会不会学错东西”，做法律 agent 时也要问“agent 会不会靠流畅文本掩盖漏召回”。

它也补充了一个维度：我不只是会做产品页面，也做过严肃计算机视觉研究，理解论文里的 ablation、metric、dataset 和 limitation 如何构成一个可信论证。这个能力迁移到 agent 产品上，就是把系统输出写成可验证报告，而不是营销文案。

10. 结论

Long-Term ReID 的贡献边界很克制：我强调数据集设计与整理，而不是把整篇 ICCV 方法都据为己有。但这段经历仍然是核心背景，因为它让我理解严肃技术项目如何通过数据、metric、ablation 和 limitation 建立可信度。
