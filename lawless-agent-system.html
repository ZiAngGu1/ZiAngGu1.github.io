01 / Legal AI

Lawless: 把法律和税务 AI 做成可交付的 agent workspace

法律/税务 AI 的核心工程，是把专家原本分散在邮件、材料、来源、草稿和审批里的工作，压成一套可输入、可检索、可验证、可审批的 harness。

真实任务

税务邮件研究、专利风险检索、TRO 冻结响应，三类任务都需要把低结构材料转成专家可复核的工作包。

核心做法

把 intake、source extraction、evidence matrix、citation preflight、draft review 和 approval gate 做成系统流程。

结果边界

税务邮件内部流程从 30-40 分钟压到约 2-3 分钟 reviewer workflow；专利内部样例 top-k recall 从约 10% 到约 90%。

1. 为什么做：专家时间要回到判断上

法律和税务工作里，最费时间的部分通常是把材料整理到可以判断的状态。税务专家要从旧邮件、客户背景、内部资料和法规来源里恢复上下文；专利顾问要从商品图片、网页、品牌、权利要求和法律状态里判断哪些专利值得复核；TRO 团队要从冻结通知、法院文件、平台截图和卖家描述里确定案件来源、资金暴露、证据缺口和下一步路径。

这些任务需要先把事实归位，再生成可复核文本。Lawless 产品化的是专家工作前面的 harness：让材料进入固定入口，让证据被拆成可检查对象，让草稿引用真实来源，让高风险动作在人工审批之后才发生。律师和税务专家因此接手的是已整理的工作包，无需重新收拾 agent 输出。

2. 怎么做：把专家工作拆成可运行的 harness

Lawless 的工作流从 intake 开始。用户上传邮件、商品页面、冻结通知或法院材料后，系统先抽取 case facts：客户是谁、任务类型是什么、已有材料有哪些、缺哪些证据、哪些来源需要引用。随后 skill routing 把任务送到税务研究、patent-find 或 ecommerce-tro-settlement，让不同法律任务进入各自的输入和输出约束。

检索和写作之间有一层 evidence harness。专利任务会形成 HIGH_RISK / RELATED bucket、claim/status/evidence matrix 和 source list；TRO 任务会形成 case brief、money exposure、evidence checklist、path cards 和 draft email；税务任务会形成 source-backed 中英文草稿和 reviewer checklist。citation preflight 检查链接、source id、正文引用和结构化 bucket，让专家把时间放在判断本身，减少先修格式和找链接的机械工作。

审批层把 agent 能做什么和系统允许做什么分开。assistant 可以读授权材料、检索来源、整理证据、生成草稿和提交 artifact；发送邮件、对外沟通、删除归档、法律承诺和最终意见必须进入 human review。每个动作都按风险级别进入不同出口，这是 harness 的一部分。

3. 结果是什么：机械整理被压缩，专家判断被保留

在企业税务场景，内部 workflow test 中，原本约 30-40 分钟的邮件研究和起草流程，被压缩成约 2-3 分钟的 reviewer workflow。专家打开任务时已经有客户上下文、来源、双语草稿和不确定点，可以直接判断、修改和批准。

在专利场景，约 100 个内部标注风险样例上，answer-blind eval 让 agent 先在不知道答案的情况下检索和报告，再由 grader 检查 high-risk recall、source support 和 evidence matrix。经过 skill/search strategy、source policy、ranking helper 和 verifier rubric 的迭代，top-k recall 从约 10% 提升到约 90%。这个结果证明 harness 能推动检索策略变好；最终法律意见仍由专家给出。

4. 边界：法律判断仍然属于专家

Lawless 的边界必须写清楚。系统能让材料更快进入可复核状态，能检查引用和结构化证据，能让草稿进入审批流，但不能把法律判断伪装成最终意见，也不能绕过律师、税务专家或运营的批准。TRO 的 shared workspace 和 case artifact 已经能展示方向，但 TRO-specific eval 仍薄于 patent search，需要继续补材料缺口识别、路径建议一致性和 draft review 质量评测。

这套工作真正特殊的地方，是把法律/税务专家每天要做的证据整理、来源核验、草稿审阅和动作审批变成可测试的工程表面。agent 输出因此变成可以被专家接手的工作包。
